# Knowledge-Share Sessions

## Suggested outline (60–75 minutes)
1. **Context (5 min)**: Why plan/apply exists for IAM and how it mirrors CaC-Ex.  
2. **Architecture (10 min)**: Review `application-pattern.md` and `diagrams/plan-sequence.mmd`.  
3. **Plan deep dive (10–15 min)**: Walk through `docs/plan-lifecycle.md`, parsing strategies, and ordering rules.  
4. **Apply deep dive (10–15 min)**: Cover appliers, API services, and `StateFileService`.  
5. **Hands-on (15–20 min)**: Edit a sample policy/role/user file, run `--plan`, inspect `masterplan.html`, then run `--apply`.  
6. **Testing & ops (5–10 min)**: Highlight `testing-steps.md`, configuration toggles, and health checks.  
7. **Q&A / actions (5 min)**: Capture doc updates or backlog items.

## Follow-ups
- Post links to the updated Confluence category pages and specs.  
- Record one run-through of the hands-on section for async onboarding.  
- Schedule refreshers when new categories or SDK behaviors are introduced.
